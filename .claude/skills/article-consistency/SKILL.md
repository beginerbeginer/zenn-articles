# 整合性チェック（consistency）

記事の中身が「食い違っていないか」を点検する。読みやすさ・誤字とは別の責務で、
ここでは **内容の正しさと一貫性** だけに集中する。

## チェック項目

### 1. フロントマター ⇔ 本文
- `title` が本文の内容を正しく表しているか。
- `topics` が本文で扱う主題を反映しているか（過不足がないか、1〜5個に収まっているか）。
- `type`（tech/idea）が記事の性質に合っているか。
- `published` が意図どおりか（下書きなら `false`）。

### 2. 事実・固有名詞・数値
- 人名・製品名・イベント名・組織名の表記が正しく、本文中で揺れていないか。
- 日付・時刻・順序（タイムライン等）が資料と一致しているか。
- 数値・計算（例：「3^5 = 243」「9通りに圧縮」）が正しいか。
- 与えられた参考資料がある場合は、それと突き合わせて裏取りする。

### 3. 見出し構造
- 見出しレベルが飛んでいないか（h2 の下にいきなり h4 等）。
- 章番号・ステップ番号が連番になっているか。
- 目次として見出しだけ読んで筋が通るか。

### 4. 用語・主張の一貫性
- 同じ概念に同じ用語を使っているか（表記の統一は校正側、ここでは概念のブレを見る）。
- 「〜と前述した」の参照先が実在するか。
- 主張に対して具体例・図表が正しく対応し、結論を支えているか。

### 5. リンク・参照
- リンク先 URL が妥当で、リンクテキストと内容が一致しているか。
- コードブロックの言語指定や、表のセル数が揃っているか。

## 進め方

- 疑わしい事実は推測で直さず、**ユーザーに確認するか TODO として明示**する。
- 修正は「どこが・なぜ食い違っていたか」を添えて報告する。
