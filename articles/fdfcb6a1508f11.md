## はじめに

Intel MacからApple Silicon Macへの移行にあたり、macOS標準のMigration Assistant（移行アシスタント）を**あえて使わず**、dotfiles管理ツール「chezmoi」を中心にクリーンセットアップする手順をまとめます。

### この記事の対象読者

- Intel MacからApple Silicon Macへの移行を予定しているエンジニア
- Migration Assistantでレガシーな設定やゴミファイルを持ち込みたくない方
- dotfilesをGit管理して再現可能な開発環境を構築したい方

### この記事で扱うこと

- chezmoi を使った移行のメリット・デメリット
- 移行前の棚卸し・スリム化手順
- chezmoi リポジトリの構築
- 新Macでのセットアップ手順
- Apple Silicon 固有の注意点

### この記事で扱わないこと

- Migration Assistantを使った移行手順
- Time Machineからの復元手順
- iCloudデータの移行（写真・メモ等）

## chezmoiによる移行のメリット・デメリット

### メリット

| 項目                     | 説明                                                                     |
| ------------------------ | ------------------------------------------------------------------------ |
| 再現性                   | dotfilesがGit管理され、いつでも同じ環境を再構築できる                    |
| クリーンスタート         | 不要な設定・ツールを持ち込まない                                         |
| アーキテクチャ差分の吸収 | テンプレート機能でIntel/Apple Siliconの差分を1ファイルで管理できる       |
| ポータビリティ           | 次回の移行や複数Mac間の同期が `chezmoi init && chezmoi apply` で完了する |
| 透明性                   | 何がインストールされ、何が設定されるか全てコードで見える                 |

### デメリット

| 項目            | 説明                                                            |
| --------------- | --------------------------------------------------------------- |
| 初期構築コスト  | dotfilesの棚卸し・テンプレート化に時間がかかる                  |
| GUIアプリの設定 | Raycast, Rectangle等のGUIアプリ設定はdotfiles外で別途対応が必要 |
| 機密情報の管理  | SSH鍵・GPG鍵・Keychain等はchezmoi単体では完結しない             |
| 学習コスト      | chezmoiのテンプレート構文やディレクトリ規約の理解が必要         |

## 前提環境

この記事では以下の環境を前提とします。

| 項目                 | 旧Mac（移行元）   | 新Mac（移行先）         |
| -------------------- | ----------------- | ----------------------- |
| チップ               | Intel             | Apple Silicon（M1以降） |
| OS                   | macOS Ventura以降 | macOS Sonoma以降        |
| シェル               | zsh               | zsh                     |
| パッケージマネージャ | Homebrew          | Homebrew                |

## Step 1: 旧Macでの棚卸し・スリム化

移行前に旧Macの環境を徹底的に棚卸しします。**ここが最も重要なステップです。**

### 1-1. Homebrewパッケージの棚卸し

まず、明示的にインストールしたパッケージ一覧を確認します。

```bash
# formulae（CLIツール）の一覧
brew leaves
```

実行例：

```
bat
chezmoi
fd
fzf
gh
git
jq
neovim
ripgrep
starship
tmux
tree
zoxide
```

次に、cask（GUIアプリ）の一覧を確認します。

```bash
# cask（GUIアプリ）の一覧
brew list --cask
```

実行例：

```
1password
docker
figma
google-chrome
iterm2
raycast
slack
visual-studio-code
warp
```

依存パッケージとして入っているだけのものを確認します。

```bash
# 自動削除可能な不要パッケージの確認（ドライラン）
brew autoremove --dry-run
```

### 1-2. 不要パッケージの削除

棚卸し結果をもとに、使っていないパッケージを削除します。

```bash
# 使っていないformulaの削除
brew uninstall <パッケージ名>

# 使っていないcaskの削除
brew uninstall --cask <アプリ名>

# 依存でしか入っていない不要パッケージの一括削除
brew autoremove

# キャッシュの削除
brew cleanup --prune=all
```

### 1-3. Brewfileの生成

スリム化が完了したら、移行用のBrewfileを生成します。

```bash
# Brewfileの生成
brew bundle dump --file=~/Brewfile
```

生成された `Brewfile` の内容を確認します。

```bash
cat ~/Brewfile
```

出力例：

```ruby
tap "homebrew/bundle"
tap "homebrew/cask"

# CLIツール
brew "bat"
brew "chezmoi"
brew "fd"
brew "fzf"
brew "gh"
brew "git"
brew "jq"
brew "neovim"
brew "ripgrep"
brew "starship"
brew "tmux"
brew "tree"
brew "zoxide"

# GUIアプリ
cask "1password"
cask "docker"
cask "google-chrome"
cask "iterm2"
cask "raycast"
cask "slack"
cask "visual-studio-code"
cask "warp"
```

:::message
**ポイント**: `brew bundle dump` で出力された内容を**1行ずつ確認**し、不要なものは削除してください。「とりあえず全部持っていく」は移行の意味がなくなります。
:::

### 1-4. dotfilesの棚卸し

管理対象とするdotfilesを洗い出します。

```bash
# ホームディレクトリ直下のdotfilesを一覧
ls -la ~/ | grep '^\.'
```

一般的に管理対象となるファイル：

```
~/.zshrc
~/.zprofile
~/.zshenv
~/.gitconfig
~/.gitignore_global
~/.config/nvim/
~/.config/starship.toml
~/.tmux.conf
~/.ssh/config        ← ホスト設定のみ（秘密鍵は含めない）
```

### 1-5. Intel固有のPATH設定を洗い出す

Apple Siliconでは Homebrew のインストール先が変わります。

| 項目           | Intel Mac           | Apple Silicon Mac      |
| -------------- | ------------------- | ---------------------- |
| Homebrewパス   | `/usr/local/bin`    | `/opt/homebrew/bin`    |
| Homebrewセラー | `/usr/local/Cellar` | `/opt/homebrew/Cellar` |

以下のコマンドで、Intel固有のパスをハードコードしている箇所を洗い出します。

```bash
# .zshrc, .zprofile 等から /usr/local を参照している行を検索
grep -rn '/usr/local' ~/.zshrc ~/.zprofile ~/.zshenv ~/.config/ 2>/dev/null
```

### 1-6. 言語ランタイムの棚卸し

バージョンマネージャで管理しているランタイムを確認します。

```bash
# asdfを使っている場合
asdf list

# miseを使っている場合
mise list

# 個別マネージャの場合
nodenv versions
rbenv versions
pyenv versions
```

:::message alert
**注意**: 移行を機にバージョンマネージャを統一することを推奨します。asdf / mise / nodenv / rbenv がバラバラに入っている場合、miseに統一するなどの判断をこのタイミングで行いましょう。
:::

### 1-7. 機密情報の確認

移行が必要な機密情報をリストアップします。

```bash
# SSH鍵の一覧
ls -la ~/.ssh/

# GPG鍵の一覧
gpg --list-secret-keys --keyid-format=long
```

## Step 2: chezmoiリポジトリの構築（旧Mac）

### 2-1. chezmoiの初期化

```bash
# chezmoiのインストール（未インストールの場合）
brew install chezmoi

# 初期化（GitHubリポジトリを新規作成しておく）
chezmoi init
```

### 2-2. dotfilesの追加

管理対象のdotfilesをchezmoiに追加します。

```bash
# .zshrcを追加
chezmoi add ~/.zshrc

# .zprofileを追加
chezmoi add ~/.zprofile

# .gitconfigを追加
chezmoi add ~/.gitconfig

# .gitignore_globalを追加
chezmoi add ~/.gitignore_global

# starship設定を追加
chezmoi add ~/.config/starship.toml

# tmux設定を追加
chezmoi add ~/.tmux.conf

# SSH config（ホスト設定のみ）を追加
chezmoi add ~/.ssh/config
```

### 2-3. chezmoiのディレクトリ構成を確認

```bash
chezmoi cd
tree -a -L 2
```

出力例：

```
.
├── .chezmoi.toml.tmpl
├── dot_gitconfig
├── dot_gitignore_global
├── dot_tmux.conf
├── dot_zprofile
├── dot_zshrc
├── private_dot_ssh
│   └── config
└── private_dot_config
    ├── nvim
    └── starship.toml
```

### 2-4. テンプレートの作成（Intel / Apple Silicon 分岐）

`.zshrc` をテンプレート化して、アーキテクチャごとにPATHを分岐させます。

```bash
# .zshrcをテンプレートに変換
chezmoi chattr +template ~/.zshrc
```

chezmoiのソースディレクトリで `dot_zshrc.tmpl` を編集します。

```bash
chezmoi edit ~/.zshrc
```

以下のようにテンプレート構文を使って分岐を記述します。

```bash
# Homebrew
{{ if eq .chezmoi.arch "arm64" -}}
eval "$(/opt/homebrew/bin/brew shellenv)"
{{ else -}}
eval "$(/usr/local/bin/brew shellenv)"
{{ end -}}

# 共通設定
export EDITOR="nvim"
export LANG="ja_JP.UTF-8"

# fzf
[ -f ~/.fzf.zsh ] && source ~/.fzf.zsh

# starship
eval "$(starship init zsh)"

# zoxide
eval "$(zoxide init zsh)"

# エイリアス
alias ls="ls -G"
alias ll="ls -la"
alias g="git"
alias dc="docker compose"
```

### 2-5. chezmoi設定ファイルのテンプレート作成

`~/.local/share/chezmoi/.chezmoi.toml.tmpl` を作成します。

```toml
[data]
    name = "Your Name"
    email = "your-email@example.com"

{{ if eq .chezmoi.arch "arm64" -}}
[data.homebrew]
    prefix = "/opt/homebrew"
{{ else -}}
[data.homebrew]
    prefix = "/usr/local"
{{ end -}}
```

### 2-6. run_onceスクリプトの作成（Homebrew一括インストール）

初回セットアップ時にBrewfileからパッケージを一括インストールするスクリプトを作成します。

```bash
chezmoi cd
```

`run_once_install-packages.sh.tmpl` を作成します。

```bash
#!/bin/bash
set -euo pipefail

# Homebrewがインストールされていなければインストール
{{ if eq .chezmoi.arch "arm64" -}}
if [ ! -f /opt/homebrew/bin/brew ]; then
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    eval "$(/opt/homebrew/bin/brew shellenv)"
fi
{{ else -}}
if [ ! -f /usr/local/bin/brew ]; then
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    eval "$(/usr/local/bin/brew shellenv)"
fi
{{ end -}}

# Brewfileからパッケージをインストール
brew bundle --file="{{ .chezmoi.sourceDir }}/Brewfile" --no-lock
```

Brewfileもchezmoiリポジトリに含めます。

```bash
cp ~/Brewfile "$(chezmoi source-path)/Brewfile"
```

### 2-7. GitHubにプッシュ

```bash
chezmoi cd
git add -A
git commit -m "feat: initial dotfiles setup"
git remote add origin git@github.com:<your-username>/dotfiles.git
git push -u origin main
```

## Step 3: 機密情報の移行準備（旧Mac）

### 3-1. SSH鍵のエクスポート

```bash
# SSH鍵を安全な場所にコピー（USBメモリやAirDrop等）
cp -r ~/.ssh /Volumes/USBDrive/ssh-backup/
```

:::message alert
**注意**: SSH秘密鍵はGitリポジトリに**絶対に含めないでください**。USBメモリ・AirDrop・1Password等で直接移行します。
:::

### 3-2. GPG鍵のエクスポート

```bash
# 秘密鍵のエクスポート
gpg --export-secret-keys --armor > /Volumes/USBDrive/gpg-secret-keys.asc

# 公開鍵のエクスポート
gpg --export --armor > /Volumes/USBDrive/gpg-public-keys.asc

# 信頼データベースのエクスポート
gpg --export-ownertrust > /Volumes/USBDrive/gpg-ownertrust.txt
```

## Step 4: 新Macのセットアップ

### 4-1. macOSの初期設定

新Macの電源を入れ、macOSの初期設定ウィザードを完了します。Migration Assistantのステップでは「今はしない」を選択します。

### 4-2. Xcode Command Line Toolsのインストール

```bash
xcode-select --install
```

ダイアログが表示されるので「インストール」をクリックします。完了まで数分かかります。

### 4-3. Homebrewのインストール

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

Apple Siliconの場合、インストール後にPATHを通す必要があります。

```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
```

インストールを確認します。

```bash
brew --version
which brew
```

期待される出力：

```
Homebrew 4.x.x
/opt/homebrew/bin/brew
```

### 4-4. chezmoiのインストールと適用

```bash
# chezmoiのインストール
brew install chezmoi

# GitHubリポジトリから初期化して適用（ワンライナー）
chezmoi init --apply <your-github-username>
```

:::message
**ポイント**: `chezmoi init --apply` は以下を一度に実行します。

1. GitHubリポジトリをクローン
2. テンプレートを評価（Apple Siliconと判定される）
3. dotfilesをホームディレクトリに配置
4. `run_once` スクリプトを実行（Brewfileからのパッケージインストール）
   :::

### 4-5. 適用結果の確認

```bash
# chezmoiが管理しているファイルの一覧
chezmoi managed

# 差分がないことを確認
chezmoi diff
```

### 4-6. SSH鍵の配置

```bash
# バックアップから復元
cp -r /Volumes/USBDrive/ssh-backup/ ~/.ssh/

# パーミッションの修正
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_*
chmod 644 ~/.ssh/*.pub
chmod 644 ~/.ssh/config

# 接続テスト
ssh -T git@github.com
```

期待される出力：

```
Hi <your-username>! You've successfully authenticated, but GitHub does not provide shell access.
```

### 4-7. GPG鍵のインポート

```bash
# 秘密鍵のインポート
gpg --import /Volumes/USBDrive/gpg-secret-keys.asc

# 公開鍵のインポート
gpg --import /Volumes/USBDrive/gpg-public-keys.asc

# 信頼データベースのインポート
gpg --import-ownertrust /Volumes/USBDrive/gpg-ownertrust.txt

# 確認
gpg --list-secret-keys --keyid-format=long
```

## Apple Silicon移行時の注意点まとめ

### PATHの変更

| 項目                     | Intel                 | Apple Silicon            |
| ------------------------ | --------------------- | ------------------------ |
| Homebrewのプレフィックス | `/usr/local`          | `/opt/homebrew`          |
| brew実行ファイル         | `/usr/local/bin/brew` | `/opt/homebrew/bin/brew` |

### Dockerイメージ

Intel向けに作られたDockerイメージはApple Siliconではエミュレーションで動作し、パフォーマンスが低下します。

```bash
# x86_64イメージを明示的に指定する場合
docker run --platform linux/amd64 <image-name>
```

可能であればマルチアーキテクチャ対応イメージ（`linux/arm64`）を使用してください。

### Homebrew Caskの互換性

一部のcaskアプリはApple Siliconにネイティブ対応していない場合があります。

```bash
# インストール済みアプリのアーキテクチャ確認
file /Applications/*.app/Contents/MacOS/*
```

出力に `x86_64` のみ表示されるアプリはRosetta 2経由で動作しています。

## トラブルシューティング

### chezmoi apply で差分が出る場合

```bash
# 差分の詳細を確認
chezmoi diff

# 特定のファイルだけ適用
chezmoi apply ~/.zshrc

# ドライランで確認してから適用
chezmoi apply --dry-run --verbose
```

### Homebrewパッケージのインストールに失敗する場合

```bash
# Homebrewの状態を確認
brew doctor

# Xcodeのパス設定を確認
xcode-select -p
# 期待値: /Library/Developer/CommandLineTools

# キャッシュをクリアして再試行
brew cleanup
brew install <パッケージ名>
```

### PATHが通らない場合

```bash
# 現在のPATHを確認
echo $PATH | tr ':' '\n'

# /opt/homebrew/bin が含まれていない場合
eval "$(/opt/homebrew/bin/brew shellenv)"

# .zprofileが正しく読み込まれているか確認
cat ~/.zprofile
```

## まとめ

Migration Assistantを使わず、chezmoiでクリーンに移行する手順を紹介しました。

### 移行手順の全体像

1. **旧Macで棚卸し・スリム化**: 不要なパッケージを削除し、Brewfileを整理
2. **chezmoiリポジトリの構築**: dotfilesをテンプレート化してGitHub管理
3. **機密情報の移行準備**: SSH鍵・GPG鍵を安全にエクスポート
4. **新Macのセットアップ**: `chezmoi init --apply` でワンコマンド構築
5. **動作確認**: 基本ツール・Docker・PATHの確認

### 次にやること

- [ ] VS Codeの拡張機能をSettings Syncで同期
- [ ] GUIアプリの設定を手動で移行（Raycast, Rectangle等）
- [ ] 旧Macのデータを完全消去

chezmoiでdotfilesを管理しておけば、次回のMac移行は `chezmoi init --apply` の一発で完了します。初期構築のコストは確かにかかりますが、一度整備すれば今後の移行コストはほぼゼロになります。
