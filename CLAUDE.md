# CLAUDE.md

このリポジトリは [Zenn](https://zenn.dev/) の記事・本を Markdown で管理するリポジトリ。
記事は `articles/<slug>.md` に1ファイル1記事で置き、Zenn CLI でプレビュー・公開する。

## 基本方針

- **本文はすべて日本語で書く。** 技術用語・コード識別子は原語のまま。
- アクセント記号付き文字（ä, ü, ã, ç 等）は ASCII に置き換えず、正しく表記する。
- 新規作成時の `published` は必ず `false`（下書き）。公開はユーザーの判断で行う。
- push は外向きの操作。**ユーザーの明示的な指示があるまで push しない。**
- コミットメッセージは Why（なぜ変更したか）を日本語で書く。How はコードを見れば分かるので書かない。

## 記事を書くときの標準ワークフロー

記事の新規作成・加筆では、責務ごとに分けた以下のスキルを順に使う。
「とりあえず書き始める」のではなく、**事前確認 → 執筆 → レビュー** の流れを毎回踏む。

1. **執筆前の事前確認** … `article-preflight`
   目的・想定読者・スタイル・type/topics/公開設定・構成案を、書き始める前に確認する。
2. **執筆 / 作成・プレビュー・公開フロー** … `zenn-article`
   フロントマター書式・slug 規則・プレビュー・公開（push）の手順。
3. **整合性チェック** … `article-consistency`
   フロントマターと本文、事実・固有名詞・日付・数値、見出し階層、用語の一貫性を点検する。
4. **読みやすさレビュー** … `article-readability`
   一文一義・結論先出し・箇条書き/表/図の活用など、構成と表現を整える。
5. **添削・校正** … `article-proofread`
   誤字脱字・表記ゆれ・文体統一・日本語の正しさを最終チェックする。

> 小さな修正では全工程を踏む必要はないが、**新規記事や大きな加筆では 3〜5 を必ず通す。**

## 機密情報

コミット・push 前に、記事に API キー・トークン・個人情報などが混入していないか確認する
（`git-secret-guard` スキルを利用してよい）。
